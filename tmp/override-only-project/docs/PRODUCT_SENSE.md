# Product Sense

## Core questions

- Which user problem is worth solving now?
- What would make the workflow feel obviously better, not just technically different?
- Which complexity belongs in the product, and which should stay hidden?

## Non-goal reminders

- Do not confuse internal flexibility with user value.
- Do not add knobs that mostly help implementers.
- Do not call a flow simple if it only becomes simple after training.

## Use this file for

- Durable product tradeoffs
- Value ranking across features
- Language the team wants to keep stable in specs and reviews
