# Execution Plan Rules

## When to open a plan

- The task needs multiple implementation steps
- More than one role needs to collaborate
- The work may span multiple sessions
- The risk is high enough that future contributors need the reasoning trail

## Plan lifecycle

1. Open a plan under `docs/exec-plans/active/`.
2. Keep scope, work plan, and validation notes current as decisions move.
3. Archive the plan to `docs/exec-plans/completed/` when the task reaches a terminal state.

## Plan quality bar

- Scope is clear enough that a Builder could start without guessing
- Validation is explicit enough that a Tester can tell what is still unverified
- Follow-up work is separated from the current task instead of quietly expanding it
