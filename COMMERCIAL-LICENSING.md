# Commercial Licensing

This repository is free for Personal Use only.

Any Commercial Use requires a separate written commercial license from the Licensor.

## What Counts As Commercial Use

Commercial Use includes, for example:

- use inside a company or organization
- use for client delivery
- use in a paid product or service
- use in internal tooling that supports business operations
- use in a hosted or managed service
- use by a team inside a startup, enterprise, agency, school, or government body

## What Usually Counts As Personal Use

Personal Use usually includes:

- learning the codebase
- trying the CLI locally as an individual
- personal non-commercial experiments
- hobby projects with no commercial objective

## How To Obtain Commercial Rights

The commercial license should be negotiated directly with the Licensor.

The Licensor may choose to provide:

- one-time project licenses
- team licenses
- subscription licenses
- OEM or redistribution licenses

## Recommended Maintainer Setup

Before public release, the maintainer should publish at least one of the following:

- a business contact email
- a website with licensing terms
- a GitHub profile or repository contact channel dedicated to licensing inquiries

## Suggested Inquiry Template

If you want to request a commercial license, include:

- company or organization name
- intended use case
- number of developers or operators
- whether redistribution is required
- expected deployment model
