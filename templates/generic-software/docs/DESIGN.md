# Design Principles

## Experience bar

- Interfaces should feel intentional, not accidental.
- Important actions must be easy to find and hard to misuse.
- Empty, loading, error, and success states must feel part of the same product.

## Review questions

- What does the user see first?
- Which element carries the decision weight on the screen?
- Does the layout reveal system state without making the user decode it?
- Is there a single strongest call to action?
