# 执行计划规则

## 什么时候需要开计划

- 任务需要多个实现步骤
- 需要多个角色协作
- 工作可能跨越多个会话
- 风险高到未来贡献者需要看到推理轨迹

## 计划生命周期

1. 在 `docs/exec-plans/active/` 下创建计划。
2. 随着决策变化，同步维护范围、工作计划和验证记录。
3. 当任务结束、取消或进入终态时，把计划归档到 `docs/exec-plans/completed/`。

## 计划质量标准

- 范围要清晰到 Builder 无需猜测就能开始
- 验证方案要明确到 Tester 能判断哪些仍未覆盖
- 后续工作应从当前任务中拆出去，而不是悄悄扩张范围
